What This Demo Shows
BEFORE
Days-to-weeks provisioning via three tools

IIP (the sole Git writer) writes tool-specific configs to Git. GitHub Actions fans out to VRA (OpenShift), Terraform (AKS), and Cloudify (Azure VMs) independently. IIP must aggregate status callbacks from each tool. No unified control plane. No live portal status.

VRA Terraform Cloudify GitHub Actions IIP callbacks
AFTER · Crossplane
~10-minute provisioning via one control plane

IIP still writes to Git — but now a Crossplane Claim YAML to Gitea. Argo CD watches Gitea and syncs the Claim to Crossplane. Crossplane reconciles ComputeInstance + StorageVolume + VolumeAttachment. Status flows directly back to IPP via the Kubernetes API. No callbacks. No tool sprawl.

Crossplane XRD Claim YAML Argo CD Gitea IIP Service Direct status
