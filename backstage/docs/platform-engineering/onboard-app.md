# Onboard App

**Audience:** Platform Engineers, Developers (with elevated access)  
**Route:** `/infra-onboarding`

---

## Purpose

The Onboard App page is the Platform Engineering team's self-service interface for registering new applications and provisioning their initial infrastructure. It complements the Backstage **Create** flow (which is developer-facing) with a more detailed, platform-engineer-focused onboarding form.

---

## Onboarding Steps

1. **Application details** — name, team, domain, and target namespace
2. **Infrastructure requirements** — resource tier (small / medium / large), database type, storage size
3. **Network policy** — select the network policy profile (internal-only, public-ingress, or custom)
4. **Security configuration** — SPIFFE identity registration, label requirements
5. **Review & submit** — generates a Crossplane Claim and Argo CD Application manifest, commits to Git via the Backstage scaffolder

### YAML Upload

Alternatively, platform engineers can upload a pre-authored `ThreeTierApp` claim YAML directly. The uploader validates the schema, shows a diff against the current cluster state, and commits the file to Git.

**Download a starter template:**  
The YAML upload component provides a downloadable example template that matches the current `ThreeTierApp` XRD schema.

---

## Post-Onboarding

After submission:
- The claim appears in the **Crossplane** page under **Active Claims**
- The Argo CD Application appears in the **GitOps** page
- The workload is visible in the **Catalog** (if `catalog-info.yaml` was generated by the template)
- Resources appear in **My Resources**
